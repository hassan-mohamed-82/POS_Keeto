# 📊 ملخص تقرير المطعم الواحد

## ✅ تم إنشاء API جديد بنجاح!

### 🔗 Endpoint:
```
GET /api/admin/reports/restaurant/:restaurantId?startDate=YYYY-MM-DD&endDate=YYYY-MM-DD
```

---

## 📋 ما يعرضه التقرير:

### 1️⃣ **معلومات المطعم**
- الاسم (عربي، إنجليزي، فرنسي)
- المعرف

### 2️⃣ **التقرير مقسم حسب نوع الطلب:**

#### **أ) Online Order** (طلبات أونلاين)
- عدد الطلبات
- إجمالي الإيرادات
- تقسيم حسب طريقة الدفع:
  - نقدي (Cash)
  - فيزا (Visa)
  - محفظة (Wallet)
- الرسوم والعمولات
- **المستحقات: لينا كام وعلينا كام**

#### **ب) Food Aggregator** (طلبات من تطبيقات أخرى)
- نفس البيانات السابقة

#### **ج) Mykeeto** (طلبات Mykeeto)
- نفس البيانات السابقة

### 3️⃣ **الإجماليات (Totals)**
- إجمالي الطلبات من كل الأنواع
- إجمالي الإيرادات
- إجمالي النقدي
- إجمالي الفيزا
- إجمالي المحفظة
- إجمالي العمولات
- **الرصيد النهائي: لينا كام وعلينا كام**

---

## 💰 فهم المستحقات (Cash Due)

### في كل نوع طلب وفي الإجماليات، ستجد:

```json
{
  "cashDue": {
    "cashCollected": "16000.00",              // النقدية المحصلة من المطعم
    "restaurantOwesToPlatform": "3200.00",    // المطعم يدين للمنصة
    "platformOwesToRestaurant": "8000.00",    // المنصة تدين للمطعم
    "netBalance": "4800.00",                  // الرصيد النهائي
    "balanceStatus": "المنصة تدين للمطعم 4800.00 جنيه"
  }
}
```

### قراءة `netBalance`:
- **موجب (+)**: المنصة تدين للمطعم → **لينا عليهم**
- **سالب (-)**: المطعم يدين للمنصة → **هما علينا**
- **صفر (0)**: لا توجد مستحقات

## 📊 ملخص الأرقام من المثال:

### حسب نوع الطلب:
| نوع الطلب | الطلبات | الإيرادات | نقدي | فيزا | محفظة | لينا/علينا |
|-----------|---------|-----------|------|------|-------|-----------|
| أونلاين | 45 | 15,000 | 9,000 | 4,000 | 2,000 | لينا 3,000 ✅ |
| تطبيقات أخرى | 20 | 8,000 | 5,000 | 2,000 | 1,000 | لينا 1,400 ✅ |
| Mykeeto | 10 | 3,000 | 2,000 | 500 | 500 | لينا 400 ✅ |
| **الإجمالي** | **75** | **26,000** | **16,000** | **6,500** | **3,500** | **لينا 4,800** ✅ |

### النتيجة النهائية:
**المنصة تدين للمطعم 4,800 جنيه** → يجب تحويل هذا المبلغ للمطعم
